IO Framework — Corrected Master Clock Formula

Executive result

1. derived:
   the correct all-epoch local clock for the IO interior is the bare FRW proper-time integral,
   not the pure dust OS cycloid.

   t_bare(z) = integral_z^infinity dz' / [(1+z') H_bare(z')]

   with

   E(z)^2 = Omega_r (1+z)^4 + Omega_m (1+z)^3 + Omega_k (1+z)^2 + Omega_Lambda.

   Equivalently, in scale factor:

   t_bare(a) = H0_bare^-1 integral_0^a da' / sqrt(Omega_r + Omega_m a' + Omega_k a'^2 + Omega_Lambda a'^4).

2. verified:
   from T_CMB = 2.7253 K and N_eff = 3.044, the exact bare radiation density is
   Omega_r,bare = 1.225914648157e-04, not the prompt-carried 9.15e-5.

   The prompt approximation is low because Omega_r depends on h^-2; with H0_bare = 58.4 the exact value is larger.

3. verified:
   the present bare-branch age is t_bare(z=0) = 19.181055510 Gyr.
   That independently reproduces the earlier ~19.2 Gyr claim.

4. verified / correction:
   once Omega_r is kept, the dust cycloid is only a matter-era approximation over a finite redshift window.
   It is not the true z -> infinity asymptote anymore.

Asymptotics

Matter-era intermediate asymptote (1 << z << z_eq):
  t_bare / t_LCDM -> H0_LCDM sqrt(Omega_m,LCDM) / [H0_bare sqrt(Omega_m,bare)]
  = 1.459211029651

Dust-cycloid matter-era asymptote:
  tau_cycloid / t_LCDM -> H0_LCDM r_s sqrt(Omega_m,LCDM) / (c x^(3/2))
  = 1.459688233857

Their ratio is
  tau_cycloid / t_bare -> 1.000327028920
so the two matter-era asymptotes differ by only 0.0327%.

Ultimate radiation-era asymptote (z >> z_eq):
  t_bare / t_LCDM -> H0_LCDM sqrt(Omega_r,LCDM) / [H0_bare sqrt(Omega_r,bare)]
  = 1.000281067483

So the user-expected 1.460 limit is not the true z -> infinity limit of the radiation-included master clock.
It is the matter-era asymptote reached across the Paper 28 table range.

Closed-form status
After the change of variable to scale factor, the exact clock is an integral of 1/sqrt(quartic).
That has no elementary primitive. It can be written in elliptic-integral form after quartic root factorization,
but numerical quadrature is cleaner and less branch-sensitive for actual IO work.
